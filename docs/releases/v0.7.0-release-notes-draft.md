# Pre-Babel Lens v0.7.0 (Draft)

## User-facing update information / ユーザー向けアップデート情報

This release stabilizes the macOS translation workflow with clearer controls, better formatting preservation, and more transparent fallback behavior. Keyboard shortcuts, menu structure, and status legends were improved so users can understand what happened during translation at a glance.
このリリースでは、macOS の翻訳ワークフローを安定化し、操作性・改行保持・フォールバックの可視性を改善しました。キーボードショートカット、メニュー構成、ステータス凡例を見直し、翻訳中に何が起きたかを一目で把握しやすくしています。

## Highlights

- Two translation modes are available on iOS
  - Standard (machine) translation and AI translation are both available.
  - Standard (machine) translation uses Apple Translation Framework.
  - AI translation uses the on-device LLM stack, Foundation Models.
  - Standard translation is now the default mode for startup stability and broader runtime availability.
  - iOS では「機械翻訳（標準翻訳）」と「AI翻訳」の2モードを利用できます。
  - 機械翻訳（標準翻訳）は Apple Translation Framework を利用します。
  - AI翻訳は on-device LLM の Foundation Models を利用します。
  - 起動安定性と利用可能性を優先し、デフォルトを機械翻訳に変更しました。
- Difference between AI translation and machine translation
  - AI translation is intended for nuanced/context-heavy output quality improvements when supported.
  - Standard machine translation is optimized for reliable baseline availability and predictable fallback behavior.
  - AI翻訳は、対応環境で文脈重視の品質改善を狙うモードです。
  - 機械翻訳は、安定したベースライン提供と明示的なフォールバック挙動を重視するモードです。

- macOS Translate menu and shortcuts
  - Added a dedicated top-level translation menu with `⌘Return` (Translate) and `⌘.` (Stop).
  - Added language-target submenu with AI/standard toggle and dynamic language list.
  - Added drag-and-drop text/file import into Source on macOS.
  - Supported file formats include plain-text files such as `.txt` and `.md`, `.docx`, and text-based `.pdf` files.
  - macOS 向けに専用の翻訳メニューを追加し、`⌘Return`（翻訳）と `⌘.`（中断）を設定しました。
  - 翻訳先言語サブメニューに、AI/機械翻訳切替と動的な言語一覧を追加しました。
  - macOS で Source へのテキスト/ファイルのドラッグ&ドロップ入力に対応しました。
  - 取り込み可能な形式は、`.txt`、`.md` などのプレーンテキスト系ファイル、`.docx`、およびテキスト抽出可能な `.pdf` です。
- Improved Translation Framework fallback reliability
  - Translation no longer stops when one segment fails; failed segments fall back to source and processing continues.
  - Added retry path for unsupported pairing using auto source (`und`) before source fallback.
  - 単一セグメント失敗で全体停止しないようにし、該当セグメントのみ原文フォールバックして継続するよう改善しました。
  - `unsupported_language_pairing` 時は、原文フォールバック前に `und` 自動ソースで再試行するようにしました。
- Better formatting and newline handling
  - Normalized incoming line endings to `\n` across URL/import/manual flow.
  - Preserved trailing/newline content from Translation Framework output.
  - URL 取り込み・共有取り込み・手入力を含め、改行コードを `\n` に正規化しました。
  - Translation Framework 出力の末尾改行を保持するよう改善しました。
- User-facing status legends (marker + reason text)
  - Added reusable legend rows next to `Completed`, including orange/blue marker styles.
  - Included reasons such as same-language untranslatable, AI-output fallback to machine translation, and unknown source language.
  - `Completed` 横に再利用可能な凡例行（マーカー + 理由文）を追加しました。
  - 同一言語で翻訳不可、AI出力失敗による機械翻訳、元言語不明などをユーザー向けに表示します。

## Notes

- Translation processing remains fully on-device.
- Some Translation Framework language resources may require system-managed downloads.
- 翻訳処理は引き続き on-device です。
- Translation Framework の一部言語は、初回にシステム管理の言語データが必要です。
